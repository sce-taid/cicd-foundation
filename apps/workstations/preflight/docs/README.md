# Preflight Module Documentation

This directory contains the authoritative technical documentation for the **Custom Image Preflight** layer.

## Authoritative Documentation Set

| Document                        | Purpose                                                                                                                 |
| :------------------------------ | :---------------------------------------------------------------------------------------------------------------------- |
| [Requirements](requirements.md) | **Product Requirements Document (PRD)**. Defines the "What" and "Why" including functional and non-functional mandates. |
| [Design](design.md)             | **Design Document (DD)**. Architectural specification and operational playbooks for building/testing.                   |

## Documentation Standards

- **Consistency**: All mandates must live in `requirements.md`. All implementation details must live in `design.md`.
- **Privacy**: The `privacy_notice.md` must be maintained for the automated build-time fragment generation.
- **Naming**: Use `snake_case.md` for any new supplemental files.
